Sheilz Coffee — Brand palette v2

Same dusty-rose identity from the logo, restructured around how UI color actually gets used: one interactive primary with proper contrast, a warm secondary accent, true neutrals for text and surfaces, and semantic colors for system states.

Why the original read "brownish"

The hue was never brown — sampling the logo's outer ring gives #B8807F, a genuinely muted rose. It looked brownish because it was likely doing double duty as both the decorative ring color and the UI's text/button color. Industry-standard practice never lets one saturated brand tone carry every job — text needs near-black neutrals, buttons need a contrast-safe shade, and backgrounds need to stay light and quiet. Splitting that single color into proper roles is what fixes the "brown" perception without changing the brand's actual pink identity.

What changed, and why it follows UI convention

1. Split one color into roles. Buttons/links use a saturated, AA-compliant #C2456A; decorative fills use the lighter #E8839E. Same family, different jobs — this is standard practice (compare how most products use a "600" shade for buttons and a "100–200" shade for tints).

2. Neutrals carry the bulk of the screen. Page background is a soft cream, cards are white, body text is a near-black brown — not pink. This is why polished apps rarely look like their brand color at first glance: color is reserved for ~10–15% of the UI (actions, highlights, status), not the whole canvas.

3. Added a second accent. The peach/orange from the logo's small flowers gives tags and secondary highlights somewhere to live, so primary pink stays meaningful as "this is clickable" instead of being reused for everything.

4. Real semantic colors. Success and error now use conventional, clearly-saturated green/red rather than forcing pink to represent both a confirmation and a problem.
